# 选题对比分析：选题1 (AgentEdu) vs 选题2 (MathAgent)

---

## 一、赛题基本信息对比

| 维度 | 选题1：AgentEdu (XH-202630) | 选题2：MathAgent (XH-202627) |
|---|---|---|
| **赛题名称** | 领域知识个性化生成与多智能体协同决策系统研究 | 基于 Intern-S1 的数学智能体设计与推理创新 |
| **核心任务** | 构建个性化教育平台 | 构建数学自动解题智能体 |
| **指定模型** | 无指定，自选 (当前用 DeepSeek) | **Intern-S1 API** (指定) |
| **产出形态** | 完整 Web 平台 (前后端 + 多 Agent) | Python 智能体 + 批量结果 + Demo |
| **评分重点** | 个性化效果 + 系统设计 + 展示效果 | **正确率 60%** + 创新性 + 展示 |
| **决赛要求** | 完整平台展示 | 多智能体协作 + 交互 Demo |

## 二、技术架构对比

### 2.1 宏观架构

```text
选题1 (AgentEdu)                              选题2 (MathAgent)
─────────────────                             ─────────────────
Vue3 前端                                      Gradio/Streamlit Demo
    ↓                                              ↓
FastAPI 后端                                   Python 主程序
    ↓                                              ↓
LangGraph 多 Agent 工作流                       LangGraph 多 Agent 工作流
    ↓                                              ↓
┌─────────────────────┐                       ┌─────────────────────┐
│ 诊断 → 路径 → 检索   │                       │ 解析 → 路由 → 规划   │
│ → 生成 → 审核 → 反馈  │                       │ → 求解 → 校验 → 解释  │
│ → 评测              │                       │ → 格式化              │
└─────────────────────┘                       └─────────────────────┘
    ↓                                              ↓
知识图谱 + RAG + SQLite                          数学工具链 + JSONL 日志
```

### 2.2 Agent 体系对比

| 功能 | 选题1 Agent | 选题2 Agent |
|---|---|---|
| **入口分析** | 诊断 Agent (分析学习画像) | 路由 Agent (题型分类) |
| **路径/规划** | 路径规划 Agent (知识图谱 DAG) | 规划 Agent (解题策略) |
| **知识获取** | 检索 Agent (RAG + FAISS) | *(无 RAG，靠 Intern-S1 内部知识)* |
| **核心生产** | 生成 Agent (讲义/代码/测验) | 求解 Agent (Intern-S1 推理) |
| **外部工具** | *(无专用工具 Agent)* | 工具 Agent (SymPy/SciPy/OR-Tools) |
| **质量控制** | 审核 Agent (内容纠偏) | 校验 Agent (三级校验) |
| **后处理** | 反馈 Agent (路径调整) | 解释 Agent + 格式化 Agent |
| **评测** | 评测模块 (幻觉率/覆盖率) | 评测模块 (正确率/耗时) |
| **Agent 总数** | 6 个 | 7 个 |

### 2.3 技术栈对比

| 技术维度 | 选题1 | 选题2 |
|---|---|---|
| **前端** | Vue 3 + Pinia + Vite | Gradio / Streamlit (轻量) |
| **后端** | FastAPI + SQLite | 纯 Python (无 Web 后端) |
| **Agent 框架** | LangGraph | LangGraph + lagent 兼容 |
| **LLM** | DeepSeek (自选) | Intern-S1 (指定) |
| **知识源** | NetworkX 知识图谱 + FAISS RAG | *(模型内部知识)* |
| **数学工具** | *(无)* | SymPy/NumPy/SciPy/OR-Tools/NetworkX |
| **数据持久化** | SQLite ORM | JSONL 日志文件 |
| **数据契约** | Pydantic | Pydantic |
| **部署** | Docker 容器化 | Python 脚本 |

## 三、核心能力对比

### 3.1 选题1 独有能力

| 能力 | 描述 |
|---|---|
| **知识图谱路径规划** | 基于 NetworkX DAG 的前置依赖递归算法，实现"因材施教" |
| **RAG 混合检索** | FAISS 向量检索 + 关键词容灾，本地知识库消除幻觉 |
| **学习者画像系统** | 多维画像 (Python/数学/ML 水平 + 偏好)，跨会话持久化 |
| **个性化讲义生成** | 根据画像差异生成不同风格/深度的 Markdown 讲义 |
| **精准缓存** | 基于 target_node 的毫秒级缓存命中 |
| **沉浸式 Web 前端** | Glassmorphism 设计 + SVG 动画 + 知识树可视化 |
| **闭环反馈** | 学情 → 生成 → 测评 → 反馈 → 路径调整 完整闭环 |

### 3.2 选题2 独有能力

| 能力 | 描述 |
|---|---|
| **题型路由** | 18 领域精准路由，不同题型走不同策略 |
| **数学工具链** | SymPy/SciPy/OR-Tools/NetworkX 全方位工具增强 |
| **三级校验** | 符号校验 + 数值校验 + 逻辑校验，多层互补保障正确率 |
| **自我修正** | 校验失败后提取错误原因，定向修正 Prompt 重推 |
| **批量处理** | 断点续跑 + 并发控制 + 进度追踪 |
| **启发式解释** | 将推理过程转化为教学级讲解 |
| **Pydantic 强约束输出** | JSON 格式合规率趋近 100% |

## 五、竞赛优劣势对比

### 5.1 选题1 优劣势

| 优势 | 劣势 |
|---|---|
| ✅ 完整产品形态，展示效果极强 | ❌ 工程量巨大，前后端都要做 |
| ✅ 知识图谱 + RAG 技术深度高 | ❌ LLM 依赖 DeepSeek，非指定模型 |
| ✅ 个性化教育场景社会意义明确 | ❌ 个性化效果难量化评测 |
| ✅ 前端展示科技感强 (Glassmorphism) | ❌ 前端维护成本高 |
| ✅ 已有大量代码基础 | ❌ 已知 Bug (AI 导师误判 off-topic) |

### 5.2 选题2 优劣势

| 优势 | 劣势 |
|---|---|
| ✅ 评分标准清晰（正确率 60%） | ❌ 正确率压力大，数学难题易失分 |
| ✅ 工程量相对可控 | ❌ 展示效果不如完整 Web 平台 |
| ✅ 使用指定模型 Intern-S1 更合规 | ❌ 受限于 Intern-S1 能力上限 |
| ✅ 技术路线清晰，边界明确 | ❌ 需要团队有数学基础 |
| ✅ 创新点自然（路由+校验+修正） | ❌ 高级数学题（拓扑/泛函）工具覆盖弱 |
| ✅ 对 AI Agent 的展现更纯粹 | ❌ 需要大量题目调试优化 |

## 七、综合建议

### 7.1 如果只能选一个

| 情况 | 推荐 | 理由 |
|---|---|---|
| 团队有数学强项成员 | **选题2** | 正确率评分标准清晰，有数学基础能冲高分 |
| 团队偏工程和前端 | **选题1** | 已有大量代码基础，展示效果好 |
| 团队人少、时间紧 | **选题2** | 工程量更可控，不需要前端 |
| 追求展示科技感 | **选题1** | 完整 Web 平台 + Glassmorphism 设计 |
| 追求评分确定性 | **选题2** | 正确率 60% 是客观评分，不主观 |
