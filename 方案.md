### 二、 你的专属战场：ML 资源生成 Agent 详细开发指南

在机器学习语境下，你的任务是将大模型变成一个“因材施教的吴恩达”。

#### 1. 重新定义你的结构化输出 (Pydantic Schema)

机器学习实训需要特定的板块。你需要强制 DeepSeek 输出以下 JSON 结构：

Python

```
from pydantic import BaseModel, Field
from typing import List

class MLCodeBlock(BaseModel):
    step_name: str = Field(description="例如：数据预处理、模型实例化、评估")
    python_code: str = Field(description="可直接执行的 Python 代码片段")
    explanation: str = Field(description="代码行级别的中文注释解释")

class MLGeneratedResources(BaseModel):
    theory_note: str = Field(description="理论讲义。需包含 LaTeX 公式（如需）及通俗易懂的类比。")
    dataset_instruction: str = Field(description="数据集长相说明，例如特征列是什么，标签是什么。")
    practice_guide: List[MLCodeBlock] = Field(description="分步的 Jupyter-style 代码实操指南。")
    graded_quiz: List[str] = Field(description="包含3道分阶测试题，形式包括选择或简答。")
```

#### 2. 你的核心代码逻辑与 Prompt 设计 (约 150 行)

作为生成 Agent，你最大的挑战是**防止模型在生成代码时发生“专业幻觉”**（比如捏造 Scikit-Learn 里不存在的超参数，或者忘记划分训练集）。

以下是你开发时的核心代码骨架和必须包含的 Prompt 策略：

Python

```
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
# ... 导入上面的 MLGeneratedResources

# 初始化大模型 (开启 JSON 模式约束)
llm = ChatOpenAI(model="deepseek-chat", api_key="你的KEY", base_url="https://api.deepseek.com/v1")
structured_llm = llm.with_structured_output(MLGeneratedResources)

def ml_generator_node(state: dict):
    # 提取上游信息
    algo = state.get("target_algorithm", "线性回归")
    diagnosis = state.get("diagnosis", "")
    knowledge = state.get("retrieved_knowledge", "")
    feedback = state.get("reviewer_feedback", "")
    
    # 动态构建系统提示词 (System Prompt)
    system_msg = """你是一位资深的机器学习算法导师。你需要根据学生的学情诊断，利用提供的参考资料，生成个性化的教学资源。
    
    【核心生成规则】：
    1. 理论讲解必须包含通俗类比。涉及数学公式时，请使用 LaTeX 格式，例如 $y = wx + b$。
    2. 代码实操必须使用 Python 3 标准库（如 Numpy, Pandas, Scikit-learn）。
    3. 严禁幻觉：禁止捏造库函数或不存在的 API 参数。
    4. 代码完整性：必须包含数据加载、预处理、模型训练和指标评估的完整闭环。"""

    # 构建用户提示词 (区分初次生成与打回重写)
    if not feedback:
        user_msg = f"""
        【目标算法】：{algo}
        【学情诊断】：{diagnosis}
        【RAG 检索知识】：{knowledge}
        
        请严格按照 JSON 格式输出理论讲义、数据集说明、代码步骤和测试题。
        如果学生数学基础弱，请在 theory_note 中减少求导推导，增加几何直觉解释。
        """
    else:
        user_msg = f"""
        你上次生成的 {algo} 教程未通过专业审核！
        【审核专家的修改意见】：{feedback}
        【原参考知识】：{knowledge}
        
        请务必虚心接受审核意见，重点修正出错的代码或不匹配的难度，并重新输出完整的资源 JSON。
        """

    # 组装并调用 API
    prompt = ChatPromptTemplate.from_messages([
        ("system", system_msg),
        ("user", user_msg)
    ])
    
    print(f"Generator Agent 正在思考 {algo} 的生成策略...")
    response = (prompt | structured_llm).invoke({})
    
    return {"generated_resources": response.model_dump()}
```

#### 3. 你的“黄金 Few-Shot”设计建议 (极大提升质量)

为了让 DeepSeek 明白你想要的“个性化”是什么级别，你需要在 Prompt 里塞入至少一个强对比的例子。

> **示例：如何教不同学生“逻辑回归 (Logistic Regression)”**
> 
> - **如果上游传入的诊断是“数学较好，但没写过代码”**：
>     
>     - **你的 Few-Shot 预期**：讲义中可以直接抛出极大似然估计推导和交叉熵损失函数 $L = - \frac{1}{N} \sum [y_i \log(\hat{y}_i) + (1-y_i) \log(1-\hat{y}_i)]$，但在代码部分，必须手把手教 `from sklearn.linear_model import LogisticRegression` 以及 `.fit()`, `.predict()` 的基本结构。
>         
> - **如果上游传入的诊断是“Python 熟练，但完全不懂微积分和矩阵”**：
>     
>     - **你的 Few-Shot 预期**：讲义中应避免出现复杂的求偏导，而是用“把 Sigmoid 函数想象成一个将其压缩到 0-1 之间的概率打分器”作为解释。代码部分可以给他安排进阶的交叉验证（Cross-Validation）或超参数网格搜索（GridSearchCV）任务。
>         

#### 4. 机器学习专属“防坑避雷”指南

在你开发这个节点时，重点防范 DeepSeek 容易犯的 3 个 ML 常见错误：

1. **数据泄露 (Data Leakage) 幻觉**：大模型写代码时，特别容易先对全量数据做标准化（StandardScaler），然后再划分 `train_test_split`。这是典型的 ML 新手错误。**对策**：在你的 System Prompt 中加入强规则：“数据缩放（Scaling）操作必须在训练集和测试集划分之后，且测试集只能使用训练集的 transform 参数。”
    
2. **维度不匹配 (Shape Mismatch)**：手写 numpy 矩阵运算或使用 PyTorch 时，大模型有时会搞混 `(N,)` 和 `(N, 1)`。**对策**：让你的审核 Agent 重点盯着维度问题，一旦发现，通过 `reviewer_feedback` 打回给你。
    
3. **废话前缀导致 JSON 解析失败**：一定确保使用 `with_structured_output` 或在 API 请求中强制设定 JSON Mode，否则大模型前面加一句“好的，我为您生成如下内容”，你的后续代码就会直接崩溃。
    

通过这样的设计，你负责的 Generator Agent 就不再是一个简单的“文本生成器”，而是一个能够根据状态机动态调整输出、甚至能自主修正代码 bug 的核心引擎。
