# AgentEdu 平台项目总览与进度说明

为了让您对整个项目有更清晰的把握，以下是本项目中**每个主要目录和文件的作用**，以及**三大核心模块（Agent、前端、后端）的当前开发进度与能力总结**。

---

## 一、 项目文件结构与作用详解

```text
agent_edu_platform/
├── agents/                     # 【核心工作流】存放所有的智能体逻辑与流转引擎
│   ├── graph.py                # LangGraph 核心编排：定义所有 Agent 的执行顺序和条件分支（例如审核不通过就重试）。
│   ├── workflow.py             # 统一外部入口：暴露 run_workflow() 函数供前端/后端/测试脚本一键调用完整流程。
│   ├── state.py & graph_state.py # 状态管理：定义智能体在流转过程中传递的“状态（上下文）”。
│   ├── diagnosis_agent.py      # 诊断智能体：分析学习者画像（背景、成绩等），判断其薄弱点并推荐难度级别。
│   ├── retrieval_agent.py      # 检索智能体：组装查询词，调用 rag_engine 检索知识库证据。
│   ├── generator_agent.py      # 生成智能体：**我刚刚重构的文件**。负责根据画像和检索知识生成讲义、代码和考题（支持 LLM 大模型与离线规则版双模式）。
│   ├── reviewer_agent.py       # 审核智能体：检查生成的资源是否合格（如代码是否有泄露、题目数是否达标等）。
│   └── feedback_agent.py       # 反馈智能体：根据用户做题正确率和反馈，决定下一步是变难、变简单还是重试。
│
├── backend/                    # 【后端接口】用于生产环境的 FastAPI 接口服务
│   ├── main.py                 # FastAPI 服务主入口（运行 uvicorn 启动的服务）。
│   └── api/                    # 存放各类路由接口（如 /learners/validate, /generation/run 等）。
│
├── frontend/                   # 【前端展示】用于比赛现场演示的 Streamlit 可视化页面
│   ├── app.py                  # Streamlit 主页面：**我刚刚重点优化的文件**。包含侧边栏控制、Agent流转图表、内容展示、画像对比等功能。
│   └── components/             # 前端 UI 组件库（如 agent_flow_view.py 用于渲染流程图）。
│
├── data/                       # 【数据源】
│   ├── knowledge_base/         # RAG 知识库：包含我们刚刚扩充的 10 多个 markdown 切片文件（如混淆矩阵、正则化等知识点）。
│   └── learner_profiles/       # 画像样例：存放 beginner.json, advanced.json 等测试用的预设学生数据。
│
├── rag_engine/                 # 【检索模块】
│   └── retriever.py            # RAG 检索器：负责读取 data 目录下的 markdown 文件，通过关键词匹配打分，提取 `KnowledgeEvidence`。
│
├── evaluation/                 # 【评测模块】
│   └── metrics.py              # 质量评测引擎：为生成的资源进行打分估算（如幻觉率、知识覆盖率、难度适配等）。
│
├── schemas/                    # 【数据协议】用于规范数据格式（Pydantic 模型）
│   ├── agent_state_schema.py   # 规定 Agent 流转过程中的数据结构。
│   ├── learner_schema.py       # 规定学习者画像长什么样。
│   └── resource_schema.py      # 规定最终输出的讲义、代码步骤、测试题长什么样。
│
├── scripts/                    # 【本地脚本】
│   └── run_demo.py             # 命令行测试脚本：不启动前端也能在终端跑完一套生成流程，用于快速验证。
│
├── tests/                      # 【自动化测试】
│   └── test_*.py               # pytest 测试用例：我们刚刚跑通的测试，确保代码不报错、逻辑不崩溃。
│
└── requirements.txt            # Python 依赖清单
```

## 二、 三大模块当前开发进度与能力

### 1. 多 Agent 协同模块 (Agent & LangGraph)
**当前进度**：⭐⭐⭐⭐⭐（核心链路已**完全打通且闭环**）
**目前能做什么**：
- **流程自动化**：已经实现了 `诊断 -> 检索 -> 生成 -> 审核纠偏(支持循环重试) -> 反馈决策 -> 评测` 的流水线闭环。
- **动态生成能力（新突破）**：系统具备了接入真实大模型（如 DeepSeek/Qwen）生成高维个性化内容的能力；同时，如果比赛现场网络不佳或 API 不可用，系统会自动切换至**高仿真的“离线规则Fallback”模式**，该模式能够根据初/中/高级画像产出完全不同复杂度的讲义和代码（如初学者讲解概念，高级者手写交叉验证 Pipeline）。
- **知识融合**：RAG 能够从扩充的十余篇核心知识点切片中提取素材（带匹配分数），并将证据无缝植入生成的教材中。

### 2. 前端演示模块 (Frontend)
**当前进度**：⭐⭐⭐⭐⭐（比赛演示级 MVP **已就绪**）
**目前能做什么**：
- **可视化调度**：直观展示各类 Agent 当前是“成功”、“重试”还是“失败”状态。
- **全方位展示**：左侧展示学习者画像与 RAG 命中的参考资料，中间展示动态渲染的 Agent 流转拓扑图和审核意见，右侧展示排版精美的讲义、逐行解释的实操代码和折叠答案测试题。
- **互动性强**：侧边栏可以直接切换不同的学生画像、模拟做题正确率，并通过“三类画像对比”弹窗，方便在演示答辩时给评委快速解释您的差异化分级策略。

### 3. 后端服务模块 (Backend)
**当前进度**：⭐⭐⭐（API 骨架**已搭建**，但尚未接入真实的前后端分离交互）
**目前能做什么**：
- **现有状态**：FastAPI 的路由（`/health`, `/generation/run`）结构和 Pydantic 协议均已定义好。
- **为什么目前只搭了骨架**：为了保证单机演示的绝对稳定性，目前的 Streamlit 前端（`frontend/app.py`）是**直接引入调用** `agents/workflow.py` 中的 Python 函数运行的，并没有通过 HTTP 请求调用后端接口。
- **后续潜力**：这种设计非常合理。如果未来（或初赛晋级后）您需要开发 Vue/React 网页端，或者是给微信小程序供电，只需要让前端请求这个现成的 FastAPI 骨架，骨架背后调用的就是我们现在写好的 `run_workflow()`。

---

## 💡 总结

目前的系统已经完全具备了**向评委做“单机离线”或“连网在线”无缝切换展示**的能力，所有的业务逻辑（从检索、生成、审核到反馈）都已经用代码跑通并经过了自动化测试验证。您可以非常安心地去撰写项目文档或直接启动 `frontend/app.py` 进行 Demo 演练。
