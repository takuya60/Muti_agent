# 表格数据理解

- 知识点：样本、特征、标签、ID列、数据泄漏字段、探索性分析

传统机器学习入门通常从表格数据开始。理解表格数据的第一步是区分“行”和“列”的含义：每一行通常是一条样本，每一列是一个变量。变量又可以分为特征、标签、ID、时间字段、辅助字段和潜在泄漏字段。特征是模型可用的输入，标签是要预测的目标。

在读取数据后，应先检查数据形状、字段类型、缺失率、类别分布和目标变量分布。这个过程比直接训练模型更重要，因为很多建模错误都来自对数据含义理解不足。例如在用户流失预测中，如果某列是“流失后客服回访次数”，它虽然和标签高度相关，但在真实预测时并不可用，属于数据泄漏字段。

实操流程：

```python
import pandas as pd

df = pd.read_csv('data.csv')
print(df.shape)
print(df.head())
print(df.info())
print(df.isna().mean().sort_values(ascending=False).head())
print(df['target'].value_counts(normalize=True))
```

教学建议：让学习者先解释每一列的业务含义，再决定是否进入模型训练。对初学者来说，“看懂表格”是机器学习实训的第一关。

易错点：
- 把目标列误放进特征。
- 把样本编号、姓名等 ID 当成普通数值特征。
- 忽略类别分布，导致后续只看 accuracy 时被多数类误导。
