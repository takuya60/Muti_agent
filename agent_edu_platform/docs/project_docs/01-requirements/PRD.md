# AgentEdu 产品需求规格书 (PRD)

> 文档版本: v1.0 | 最后更新: 2026-06-22

---

## 一、产品定位

**AgentEdu** 是一个基于多智能体协同与知识图谱驱动的个性化机器学习实训平台。系统通过"画像诊断 → 路径规划 → 知识检索 → 内容生成 → 质量审核 → 反馈调整"六步流水线，为不同基础的学习者自动生成量身定制的讲义、代码实操和分阶测验。

---

## 二、目标用户

| 用户类型 | 特征描述 |
|---------|---------|
| 零基础入门者 | Python 基础薄弱，数学恐惧，需要大量类比和引导 |
| 有一定基础的学习者 | 会 Python + NumPy/Pandas，想系统学机器学习 |
| 进阶工程师 | 有项目经验，想深入模型原理和调参技巧 |

---

## 三、核心功能列表

### 3.1 画像采集与诊断
- **F-001**: 用户填写结构化画像表单（背景、目标、Python/数学/ML水平、偏好风格等）
- **F-002**: 系统自动诊断薄弱点、确定推荐学习级别、制定解释策略

### 3.2 学习路径规划
- **F-003**: 基于知识图谱 (DAG) 计算个性化学习路径
- **F-004**: 树状可视化展示（主干 + 分支），支持分支切换
- **F-005**: LLM 可动态注入图谱中不存在的新节点

### 3.3 内容生成
- **F-006**: 多 Agent 协同生成个性化讲义（Markdown 格式）
- **F-007**: 生成代码实操指南（带步骤解释和预期输出）
- **F-008**: 生成分阶测验题（基础/标准/进阶三级）
- **F-009**: RAG 检索本地知识库，引用真实证据避免幻觉

### 3.4 质量审核
- **F-010**: 代码硬规则检查（如数据泄漏检测）
- **F-011**: LLM 语义审核（结构完整性、逻辑连贯性）
- **F-012**: 审核未通过自动打回重做，最多重试 3 次

### 3.5 交互问答
- **F-013**: 基于当前讲义上下文的智能问答（支持流式输出）
- **F-014**: 离题检测，拒绝与课程无关的问题

### 3.6 反馈与进阶
- **F-015**: 根据测验准确率和用户反馈决定下一步行动
- **F-016**: 生成降维补充 / 同难度巩固 / 进阶挑战资源

## 四、非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 首次生成耗时 ≤ 60s（含 LLM 调用），缓存命中时 ≤ 200ms |
| 可用性 | LLM 不可用时自动降级为规则引擎，系统 100% 可用 |
| 安全性 | API Key 不落前端，所有 LLM 调用在后端完成 |
| 可观测性 | 每次 LLM 调用自动写入本地 Trace 日志 |

## 五、用户故事

### US-001: 零基础学习者第一次进入系统
> 作为一名 Python 初学者，我填写画像后，希望系统能告诉我应该从哪里开始学，而不是直接跳到高级内容。

### US-002: 中级学习者切换学习方向
> 作为一名已经掌握基础的学习者，我完成了分类预测分支后，希望能切换到"模型可解释性"分支继续学习。

### US-003: 审核发现问题后自动修复
> 作为系统的使用者，我希望 Agent 生成的内容如果有逻辑问题，系统能自动发现并重新生成，而不需要我手动刷新。
