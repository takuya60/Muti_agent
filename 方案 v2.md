| 决策项       | 选择                                     |
| --------- | -------------------------------------- |
| 交互形态      | 混合模式：左侧对话 + 右侧资源面板                     |
| 前端        | Vue 3 + Vite + Element Plus            |
| 后端        | FastAPI（保留，补全能力）                       |
| 数据库       | SQLite（SQLAlchemy ORM，后续可切 PostgreSQL） |
| 流式输出      | SSE                                    |
| LangGraph | 事件驱动循环图                                |
| 会话        | 跨会话累积                                  |
| 对话存储      | 独立存数据库，State 只放摘要                      |
| RAG       | 向量检索 + 知识库扩充 + 知识图谱，三者同时做              |

## 三、双层 State 设计

### 3.1 持久层：学习者档案（数据库 ORM 模型）

```python
# backend/models.py

class Learner(Base):
    """学习者表：跨会话持久存储"""
    __tablename__ = "learners"
    id = Column(String, primary_key=True)           # learner_id
    name = Column(String, nullable=False)
    background = Column(Text)
    goal = Column(Text)
    preferred_style = Column(String, default="案例驱动")
    test_scores = Column(JSON, default={})           # {"python": 80, ...}
    known_skills = Column(JSON, default=[])
    weak_points = Column(JSON, default=[])
    mastered_points = Column(JSON, default=[])        # 【新增】已掌握知识点，跨会话累积
    current_level = Column(String, default="beginner_plus")  # 【新增】当前推荐难度
    created_at = Column(DateTime, default=func.now())
    updated_at = Column(DateTime, onupdate=func.now())

class Session(Base):
    """学习会话表：一次完整的学习交互"""
    __tablename__ = "sessions"
    id = Column(String, primary_key=True)            # session_id (UUID)
    learner_id = Column(String, ForeignKey("learners.id"))
    target_algorithm = Column(String)                 # 本次学习目标
    current_phase = Column(String, default="init")    # init|diagnosing|learning|quizzing|completed
    diagnosis_summary = Column(JSON)                  # 诊断结果快照
    generated_resources = Column(JSON)                # 生成的资源快照
    evaluation = Column(JSON)                         # 评测结果快照
    created_at = Column(DateTime, default=func.now())

class Message(Base):
    """对话消息表：独立存储，不放在 LangGraph State 里"""
    __tablename__ = "messages"
    id = Column(Integer, primary_key=True, autoincrement=True)
    session_id = Column(String, ForeignKey("sessions.id"))
    role = Column(String)                             # user | assistant | system | agent_event
    content = Column(Text)
    agent_name = Column(String, nullable=True)        # 如果是 agent_event，记录哪个 Agent
    created_at = Column(DateTime, default=func.now())

class QuizAttempt(Base):
    """答题记录表"""
    __tablename__ = "quiz_attempts"
    id = Column(Integer, primary_key=True, autoincrement=True)
    session_id = Column(String, ForeignKey("sessions.id"))
    learner_id = Column(String, ForeignKey("learners.id"))
    question = Column(Text)
    learner_answer = Column(Text)
    correct_answer = Column(Text)
    is_correct = Column(Boolean)
    knowledge_point = Column(String)                  # 对应知识点，用于更新 mastered_points
    created_at = Column(DateTime, default=func.now())
```

### 3.2 会话层：LangGraph 流转用的 State

```python
# agents/graph_state.py（改造后）

class AgentGraphState(TypedDict, total=False):
    # === 会话标识 ===
    session_id: str
    learner_id: str

    # === 学习者上下文（从数据库加载，不是完整画像） ===
    learner_profile: dict
    mastered_points: list[str]          # 【新增】从 Learner 表加载
    current_level: str                   # 【新增】从 Learner 表加载

    # === 当前轮次上下文 ===
    target_algorithm: str
    current_phase: str                   # 【新增】当前阶段
    user_message: str                    # 【新增】用户最新一条消息
    conversation_summary: str            # 【新增】对话历史摘要（不是全部消息）

    # === Agent 产出 ===
    diagnosis: DiagnosisResult
    retrieved_knowledge: list[KnowledgeEvidence]
    generated_resources: dict
    reviewer_feedback: str
    review_passed: bool
    feedback_decision: dict
    evaluation: dict
    agent_events: list[dict]
    retry_count: int

    # === 路由控制 ===
    next_action: str                     # 【新增】路由 Agent 的判断结果
    response_text: str                   # 【新增】要回复给用户的文本
```

> [!NOTE]
> 关键设计：`conversation_summary` 只存对话摘要（例如"用户已学完逻辑回归基础，正在做测试题，答对了2题答错了1题"），而不是存所有历史消息。完整消息存在 `messages` 数据库表中。这样 State 不会随着对话变长而膨胀。

## 四、事件驱动 LangGraph 工作流

### 4.1 流程图

```text
                          ┌─────────────┐
                          │   START     │
                          └──────┬──────┘
                                 ↓
                    ┌────────────────────────┐
                    │   init_session_node    │  加载学习者档案，初始化会话
                    └────────────┬───────────┘
                                 ↓
                    ┌────────────────────────┐
                    │   diagnosis_node       │  学情诊断
                    └────────────┬───────────┘
                                 ↓
                    ┌────────────────────────┐
                    │   retrieval_node       │  RAG 检索
                    └────────────┬───────────┘
                                 ↓
                    ┌────────────────────────┐
                    │   generation_node      │  资源生成
                    └────────────┬───────────┘
                                 ↓
                    ┌────────────────────────┐
                    │   review_node          │  审核纠偏
                    └────────────┬───────────┘
                                 ↓
                  ┌──── route_after_review ────┐
                  ↓                            ↓
            review_passed=False          review_passed=True
            且 retry < MAX                     ↓
                  ↓               ┌────────────────────────┐
            generation_node       │   evaluate_node        │  评测
                                  └────────────┬───────────┘
                                               ↓
                                  ┌────────────────────────┐
                                  │   respond_node         │  生成首条导学回复
                                  └────────────┬───────────┘
                                               ↓
                              ┌─────────────────────────────────┐
                    ┌────────→│   wait_for_user_input           │←────────┐
                    │         │   (interrupt_before)             │         │
                    │         └──────────────┬──────────────────┘         │
                    │                        ↓                            │
                    │         ┌────────────────────────┐                  │
                    │         │   router_node          │  路由 Agent      │
                    │         │   判断用户意图          │                  │
                    │         └──────────────┬─────────┘                  │
                    │                        ↓                            │
                    │         ┌──── route_by_intent ────┐                 │
                    │         ↓              ↓          ↓                 │
                    │    "ask_question" "submit_quiz" "next_topic"        │
                    │         ↓              ↓          ↓                 │
                    │    answer_node    quiz_node   new_topic_node        │
                    │         │              │          │                 │
                    │         └──────────────┴──────────┘                 │
                    │                        ↓                            │
                    │              respond_node ──────────────────────────┘
                    │                        ↓
                    │                  (如果 intent == "end")
                    │                        ↓
                    │                       END
                    └─────────── (循环回 wait_for_user_input)
```

### 4.2 各节点职责说明

| 节点 | 职责 | 触发时机 |
|---|---|---|
| `init_session_node` | 从数据库加载学习者档案，初始化 State | 会话开始时，仅一次 |
| `diagnosis_node` | 学情诊断（LLM + 规则混合） | 会话开始时，仅一次 |
| `retrieval_node` | 向量 RAG 检索知识库 | 生成资源前 |
| `generation_node` | LLM 生成讲义/代码/题目（带 fallback） | 首次 + 每次换 topic |
| `review_node` | LLM 审核（带规则 fallback） | 生成后 |
| `evaluate_node` | 质量评测 | 审核通过后 |
| `respond_node` | 组装回复文本，写入消息数据库 | 每次要回复用户时 |
| `wait_for_user_input` | **中断点**：暂停图执行，等待用户发消息 | 每次回复后 |
| `router_node` | **路由 Agent**（LLM）：判断用户意图 | 收到用户消息后 |
| `answer_node` | 回答用户追问（基于 RAG + 对话上下文） | 用户提问时 |
| `quiz_node` | 处理用户答题，更新 mastered_points | 用户提交答案时 |
| `new_topic_node` | 切换到下一个知识点，触发新一轮检索+生成 | 用户想学新内容时 |

### 4.3 路由 Agent 的意图分类

`router_node` 使用 LLM 将用户消息分类为以下意图：

| 意图 | 示例用户消息 | 后续动作 |
|---|---|---|
| `ask_question` | "Sigmoid 函数为什么要压缩到0-1？" | → `answer_node` 用 RAG 回答 |
| `submit_quiz` | "我选 A" / "答案是因为数据泄露" | → `quiz_node` 判断对错 |
| `request_hint` | "给我个提示" | → `answer_node` 给出引导 |
| `too_hard` | "太难了，能简单点吗？" | → `feedback_node` → 降维重新生成 |
| `too_easy` | "太简单了" | → `feedback_node` → 进阶挑战 |
| `next_topic` | "我想学交叉验证" | → `new_topic_node` |
| `end` | "够了，谢谢" | → END |
| `general_chat` | "你好" / 无明确意图 | → `answer_node` 自由回复 |

SSE 流的事件类型设计：

```text
event: agent_start
data: {"agent": "路由 Agent", "status": "running"}

event: agent_done
data: {"agent": "路由 Agent", "status": "completed", "summary": "判断为追问类型"}

event: token
data: {"content": "逻辑回归"}     // 流式文本输出，逐 token 推送

event: resource_update
data: {"type": "theory_note", "content": "..."}    // 资源面板内容更新

event: quiz
data: {"question": "...", "level": "基础"}         // 推送测试题

event: done
data: {}                                            // 本轮对话完成
```

### 7.1 向量检索

```python
# rag_engine/embedding.py
```
